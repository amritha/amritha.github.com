Miso Quips Creation

Relevant Skills:

Interaction Design, User Research, Visual Design

Background Info:

The SideShow creation tool did not get the chance to launch because Miso pivoted and wanted to streamline towards an entirely mobile experience. Instead of entire SideShows being created by users on the web, we wanted users to create individual text-based “quips” on their iPhones:

Sketching and Wireframing:

I began by sketching out all sorts of layouts for the creation screen itself.

A big challenge was figuring out how to adapt the image search flow for a mobile experience. The goals in this case were slightly different from the Web SideShow Creation Tools:

  1. Provide a simple way for users to upload or find a photo on the iPhone.
  2. Accelerate image search flow through prompting.
  3. Provide a simple way to scan a large amount of image search results.
  4. Allow a preview of the full image before selection.

I outlined a straightforward user flow and proceeded to simplify it from there. I benchmarked other mobile apps like Google Image Search and Photobucket for inspiration.

After a multiple iterations on my sketches, I created wireframes to finalize the interactions.

After making sure all details were ironed out for engineering implementation, I set forth on the visuals based on the existing creative direction.


I was able to tackle all of my goals in the following ways:

Simple upload from creation screen

Search suggestions based on TV Show

Grid of search results for easy scanning

Full image preview before selection
