Miso SideShow Creation Tools

Relevant Skills:

Interaction Design, User Research, Visual Design

Background Info:

When I joined Miso, I was given the project of creating our first ever SideShow Creation Tool. A Miso SideShow is a iPhone slideshow created by a user with individual pieces of content (cards) tagged to specific times in an episode of television. These cards must contain at least one image along with some text.

Check out Miso's homepage for some SideShow examples on the web.

My first task was taking the creation process from an Excel Spreadsheet to a bare minimum web interface that could be implemented by engineering in two weeks. This iteration was extremely basic, but it tackled the crucial needs of allowing image upload, card category selection, and more traditional text entry.

The Research:

The subsequent iterations were heavily influenced by user research. I spent the next 2 weeks observing all of our “TV Superfans” creating SideShows with our tool so I could tease out the problem areas.

Organizing my findings on Post-it notes helped me identify the top user pain points:

  1. Difficult to find the right image for a card
  2. No ability to crop/resize an image within the tool
  3. No ability to preview cards before publishing a SideShow
  4. Users often drop off before publishing an entire SideShow

Sketching and Wireframing:

Keeping the problem areas in mind, I created a flow diagram of all the possible steps that a user could take in the tool. Then I kicked off brainstorming by evaluating all the opportunities that I had to minimize the number of steps necessary.
